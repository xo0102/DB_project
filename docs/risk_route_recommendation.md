# 위험 기반 경로 추천 기능 설계

## 구현 목적

TMAP이 반환한 기본 보행 경로를 Supabase의 위험 데이터와 비교하여, 단순 최단 경로가 아니라 위험 근거를 설명할 수 있는 경로 추천 기능을 제공한다.

## 대안 경로 탐색 조건

다음 중 하나라도 만족하면 대안 경로를 추가 탐색한다.

1. 기본 경로의 총 위험도가 50점 이상인 경우
2. 총 위험도가 50점 미만이어도 현재 활성 상태인 최근 사용자 신고가 경로에 포함된 경우

위 조건을 만족하지 않으면 TMAP 기본 경로 1개만 표시한다.

## 반영 데이터

- `report_risk_zones` + `user_reports`: 최근 신고 위치, 반경, 위험 유형, 신고 설명, 신고 시각, 중복 신고 수
- `flood_zones`: 침수 이력·예상 구역의 중심 좌표와 기본 점수
- `road_alerts`: 활성 도로 통제·위험 알림
- `weather_snapshots`: 최신 강수 위험 점수

## 현재 위험 판별 방식

PostGIS 도입 전 단계이므로 각 위험 중심점과 TMAP 경로선 사이의 최소 거리를 계산한다.

- 최근 신고: `report_risk_zones.radius_m` 이내
- 침수 구역: 중심점에서 경로까지 100m 이내
- 도로 알림: 중심점에서 경로까지 80m 이내
- 날씨 위험: 모든 경로에 공통 반영

경로선과 점 사이 거리는 각 선분에 위험 중심점을 투영하여 근사 계산한다.

## 대안 경로 생성 방식

TMAP 기본 경로에 포함된 최근 신고와 주요 위험 위치를 선택하고, 원래 경로 진행 방향의 좌·우에 우회 경유점을 생성한다.

1. 기본 경로 분석
2. 회피할 위험 위치 최대 3개 선택
3. 위험 반경 바깥쪽에 좌·우 경유점 생성
4. TMAP `passList`를 이용하여 최대 4개 후보 경로 요청
5. 최근 신고 포함 여부, 50점 미만 여부, 총 위험도, 거리 순으로 후보 비교
6. 가장 안전한 후보를 대안 경로로 표시

## 화면 표시

- 기본 경로: 위험 조건 충족 시 빨간색
- 위험 회피 대안 경로: 초록색
- 최근 신고·침수·도로 위험 위치: 지도 원과 마커
- 경로 카드: 거리, 예상 시간, 총 위험도, 포함된 최근 신고 수
- 상세 표: 위험 출처, 위험 점수, 경로와의 거리, 신고 설명, 신고 시각, 누적 신고 수

## 현재 단계의 한계

- 중심 좌표와 반경 기반 근사 분석이므로 실제 Polygon 교차와 차이가 있을 수 있다.
- 임의 경유점이 보행 가능한 길과 멀면 TMAP이 대안 경로를 만들지 못할 수 있다.
- 대안 후보가 모두 위험 구역을 통과하면 완전 회피 성공으로 표시하지 않는다.
- 검색 결과는 아직 `route_search_logs`, `route_results`, `route_risk_details`에 저장하지 않는다.

## 다음 단계

4. 침수 GeoJSON을 Polygon 또는 MultiPolygon으로 변환
