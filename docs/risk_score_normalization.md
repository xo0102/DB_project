# 경로 위험 점수 정규화

## 문제

동일한 중심 좌표에 이름만 다른 침수 샘플 행이 여러 개 존재하면 PostGIS의
`ST_Intersects`는 각 행을 모두 반환한다. 기존 코드는 반환된 행의 점수를 단순 합산한 뒤
최종 합계만 100점으로 제한했기 때문에, 서로 다른 경로가 모두 100점으로 포화되는 문제가 있었다.

## 해결 방식

### 1. 동일 위치 위험 통합

침수 위험은 위도·경도를 소수점 다섯 자리까지 반올림한 위치를 기준으로 그룹화한다.
동일 위치에 여러 행이 있으면 최고 점수 한 번만 반영하고, 상세 설명에는 원본 행 ID와
통합 건수를 남긴다.

### 2. 유형별 점수 상한

| 유형 | 최대 반영 점수 |
|---|---:|
| 침수 이력·예상 구역 | 50 |
| 도로 통제·알림 | 20 |
| 최근 사용자 신고 | 10 |
| 날씨 | 20 |

유형별 상한의 합은 100점이다. 최근 사용자 신고는 점수가 10점을 넘지 않더라도
그 존재 자체로 대안 경로 탐색 조건을 만족한다.

### 3. 원본 점수와 반영 점수 구분

화면에는 `원본 점수`와 `반영 점수`를 함께 표시한다. 상한으로 일부 점수가 제외되면
위험 상세 사유와 `route_risk_details.reason`에 정규화 내역을 기록한다.

## 기존 데이터

이 패치는 새 경로 검색부터 적용된다. 이미 저장된 `route_results.total_risk_score`는
과거 계산 결과이므로 자동으로 변경하지 않는다.

현재 샘플 침수 데이터 자체를 정리하려면 Supabase SQL Editor에서 다음 파일을 1회 실행한다.

```text
sql/cleanup_duplicate_flood_zones.sql
```
